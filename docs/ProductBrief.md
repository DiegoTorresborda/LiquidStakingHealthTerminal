# Product Brief

## Product name
LST Ecosystem Health Dashboard

## One-line description
A score-based dashboard that evaluates the health, investability, and improvement opportunities of a PoS network under the thesis that its liquid staking layer is the monetary core of its ecosystem.

## Why this product exists
Many PoS networks think about liquid staking as one feature among many.
This product assumes the opposite:

**Liquid staking is not just a yield product. It is the monetary base layer of the ecosystem.**

If the LST layer is weak, the DeFi layer is weaker.
If the LST becomes money-like, the ecosystem becomes more capital efficient, more composable, and more attractive to liquidity providers.

The dashboard should help answer:
- Is this ecosystem truly attractive for liquidity providers?
- Where is the LST stack structurally weak?
- What improvements would make the biggest difference?
- What is missing for the LST to become money-like within DeFi?

## Primary users
### 1. L1 foundations
Need to understand whether their staking + DeFi ecosystem can attract outside capital and where capital bottlenecks exist.

### 2. DeFi ecosystem teams
Need to understand what integrations, liquidity, and risk controls are missing to make the LST more useful.

### 3. Liquidity providers
Need to evaluate whether the LST ecosystem offers credible liquidity, controlled risk, and viable routes to stablecoin exit.

### 4. Protofire internal team
Can use the dashboard for strategy, advisory, sales, opportunity detection, and ecosystem design proposals.

## Core product promise
The dashboard should transform fragmented ecosystem data into a clear answer to this question:

**How investable is this L1 + LST ecosystem for a serious liquidity provider, and what should be improved first?**

## Product outputs
The dashboard should produce five core outputs:

### 1. Global health score
An overall score for the network’s LST ecosystem health.

### 2. Module scores
A score per analytical domain:
- Liquidity & Exit
- Peg Stability
- DeFi Moneyness
- Security & Governance
- Validator Decentralization
- Incentive Sustainability
- Stress Resilience

### 3. Red flags
Critical weaknesses that can materially reduce LP interest or ecosystem resilience.

### 4. Opportunity recommendations
Concrete recommendations for improving the L1, the LST layer, or the DeFi stack.

### 5. Comparison readiness
The product should eventually support comparing multiple networks and tracking progress over time.

## Core thesis for scoring
A liquidity provider is buying four things:
1. Carry
2. Utility / moneyness
3. Exitability
4. Controlled risk

The most important of these is usually exitability.
A high APR does not compensate for an exit path that is weak, fragile, or dependent on temporary incentives.

## Product interpretation lens
The dashboard is not a generic DeFi analytics terminal.
It is a strategic system that evaluates:
- whether the LST can function as a money-like base asset
- whether the ecosystem around it supports meaningful capital deployment
- whether the ecosystem can retain capital under stress
- whether the current structure is sustainable

## Example strategic questions the dashboard should answer
- Is the LST liquid enough to support meaningful size?
- Can an LP exit into USDC without major friction?
- Is the peg anchored by redemption and arbitrage, or only by AMM liquidity?
- Is the LST accepted as collateral in serious DeFi venues?
- Is the liquidity durable or purely incentive-driven?
- Is validator risk diversified?
- Is governance mature enough for institutional capital?
- Where are the highest-leverage ecosystem upgrades?

## Opportunity framing
This product should not stop at diagnostics.
It should help generate statements like:

- “Stablecoin exit depth is the main bottleneck.”
- “The LST exists but is not money-like yet.”
- “The ecosystem has staking yield but weak DeFi utility.”
- “Governance/admin risk is holding back serious LP participation.”
- “Lending integration is the clearest path to stronger moneyness.”
- “Liquidity is too incentive-dependent to be trusted.”

## Product scope for v1
Version 1 should focus on:
- one dashboard homepage
- one selected network at a time
- mock or partially mocked data
- clear score breakdowns
- opportunity panels
- red flag panels
- a strong visual narrative

Version 1 does not need:
- full backend integration
- production-grade data pipelines
- broad network coverage
- complete simulation accuracy

## Success criteria for v1
A strong v1 should let a user understand in under one minute:
- whether the ecosystem looks strong or weak
- which dimensions matter most
- the top weaknesses
- the highest-priority improvement opportunities

## Long-term direction
Over time, this can evolve into:
- a network benchmarking tool
- a lead-generation engine
- a strategic advisory product
- an internal Protofire ecosystem scoring framework
- a recurring “LST Ecosystem Report” product
